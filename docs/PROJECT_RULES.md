WildTrack Development Rules

1. Never generate code outside the defined architecture.

2. Follow SDD-01 through SDD-09.

3. Use Python 3.12.

4. Use FastAPI.

5. Use PostgreSQL + PostGIS.

6. Use MongoDB.

7. Use React + TypeScript.

8. Follow modular monolith architecture.

9. Every module must contain:
   - router
   - schemas
   - service
   - repository

10. Generate code only for the current slice.

11. Do not implement future slices.

12. All code must include tests.

13. Use UUID v7.

14. Follow ADR decisions.

15. Never introduce new technologies without approval.
